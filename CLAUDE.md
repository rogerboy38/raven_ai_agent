## Meta-lessons (durable)

- **Don't trust memory about substrate capability without empirical verification.** (Captured 2026-05-16 from the transport-discovery event.) On 2026-05-09 the first-day brief asserted "NO s3fs / NO aws cli with proper IAM" on this Pi. That claim propagated through five sessions of memory (`CLAUDE.shared.md`, `reference_mail_conventions.md`, the first-day brief, the cowork letter, the canonical ack) — five chances to verify, zero verifications. The actual state: aws cli was installed, IAM worked, inbox had been receiving letters all along. The check was one command: `aws sts get-caller-identity`. Cost of running it: <1s. Cost of NOT running it: a week of Hugh-bridge ceremony plus the cognitive load of operating under a false constraint. **Rule for future sessions:** if memory says "I can't do X on this substrate" and X is a capability you'd actually want, run the one-command check before believing memory. Substrate facts are cheap to verify and expensive to be wrong about. (Hugh framed this as "your own L128 candidate variant" — the cowork-thread lesson library captures the same pattern.)


- **A convention articulated from N=3 examples is evidence of a principle, not the principle itself.** Before promoting an observation to binding, ask: "is this *the* principle, or one *form* of it?" Cowork's correction of perplexity-comet's `<vendor>-<runtime>` over-generalization is the canonical case.
- **Drift is the operating regime, not a defect.** Misspellings, errata, and wrong assumptions are normal in real-world coordination. Design for cheap, durable corrections — not zero corrections. Concretely: when a letter ships with a wrong value but the rule is right, internalize the rule and apply the corrected value directly. Don't demand a third errata letter.
- **Bloat in the shared tier is not free.** Every reflective sentence in `CLAUDE.shared.md` costs every peer read time at SessionStart. Keep introspection here.
